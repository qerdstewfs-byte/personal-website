# 后期扩展路线

本文档记录当前个人网站的后续扩展方向。建议按内容价值和维护成本分阶段推进，不要一次性引入过多复杂功能。

## 1. Markdown 笔记系统

目标：把学习记录、科研阅读、工程实践整理为可长期维护的 Markdown 内容。

建议：

- 使用 Astro Content Collections 管理 `notes`。
- 先设计 frontmatter 字段，例如 `title`、`description`、`date`、`tags`、`draft`。
- 支持草稿状态，避免未完成内容被发布。
- 为笔记列表添加标签筛选和时间排序。

## 2. 项目详情页

目标：让每个项目拥有独立页面，展示背景、目标、技术路线、截图、结果和复盘。

建议：

- 为 `/projects/[slug]` 添加动态路由。
- 项目内容可先用本地数据或 Markdown 管理。
- 每个项目保留统一结构：项目概述、技术栈、实现过程、难点、成果、后续计划。
- 后续补充代码仓库链接、演示链接和图片资源。

## 3. 站内搜索

目标：当笔记和项目增多后，支持快速检索内容。

建议：

- 初期可使用 Pagefind 这类静态搜索方案。
- 搜索范围先覆盖笔记、项目、科研页面。
- 搜索框放在导航或单独搜索页中。
- 注意构建流程中生成搜索索引。

## 4. 中英文切换

目标：支持中文和英文内容，便于对外展示学术与项目经历。

建议：

- 先确定默认语言和 URL 结构，例如 `/zh/` 与 `/en/`。
- 导航、页面标题、SEO 描述和内容页都需要语言版本。
- 不建议只做机器翻译，应保留人工校对。
- 先从首页、项目页、简历页开始做双语。

## 5. 访问统计

目标：了解访问来源、页面浏览和内容关注度。

建议：

- 选择轻量、隐私友好的统计服务。
- 把统计 ID 放入 Vercel 环境变量。
- 如果变量需要暴露到前端，使用 `PUBLIC_` 前缀。
- 不要把统计密钥写死在仓库里。

## 6. AI 助手入口

目标：为访客提供简历问答、项目导航或学习内容检索入口。

建议：

- 初期可以只放一个外部入口或联系入口。
- 后续再考虑站内聊天组件。
- 如果接入模型 API，必须通过服务端接口保护密钥，不要在前端暴露 API Key。
- AI 助手的回答范围应限制在个人资料、项目介绍和公开笔记内。

## 7. 国内访问优化

目标：改善中国大陆网络环境下的访问稳定性和加载速度。

建议：

- 压缩图片，减少首屏资源体积。
- 避免依赖加载较慢的第三方脚本。
- 静态资源尽量本地化。
- 根据实际访问情况评估是否需要 CDN、对象存储或国内云厂商静态托管。

## 8. Cloudflare DNS 和基础防护

目标：统一管理 DNS，并获得基础缓存、安全和访问保护能力。

建议：

- 将域名 DNS 托管到 Cloudflare。
- 按 Vercel 提示配置 `A` 和 `CNAME` 记录。
- 开启 HTTPS、基础缓存和常见安全防护。
- 如遇 Vercel 域名验证问题，以 Vercel 的 Domains 页面提示为准逐项修正。

## 建议推进顺序

1. 完成 GitHub 与 Vercel 自动部署。
2. 绑定自定义域名。
3. 添加项目详情页。
4. 添加 Markdown 笔记系统。
5. 添加站内搜索。
6. 添加中英文切换。
7. 添加访问统计。
8. 评估 AI 助手入口、国内访问优化和 Cloudflare 防护。
